代码开始自融资

代码代理终于走到了"自己给自己讲故事"的阶段 —— ARR 上 4.9 亿、企业用量月增 50%、9 成代码由 AI 自写。Devin 这一轮的估值跳跃，是整条赛道在 2026 上半年最干净的一笔注脚。

💰 头条 · COGNITION × DEVIN

Cognition 完成 $1B D 轮 · 估值跳到 $26B · Devin 写了自家 90% 的代码

5 月 27 日，Cognition 宣布完成 10 亿美元 D 轮融资，投后估值定格 260 亿美元，由 Lux Capital、General Catalyst、8VC 共同领投，Ribbit Capital、Atreides、Layer Global 等新投资人加入，Founders Fund、Elad Gil、Alpha Wave、Bain Capital Ventures 等老股东跟投。距离去年 9 月 102 亿的 C 轮仅 9 个月，估值翻了 2.5 倍。叙事的支点是三个数字：Devin 年化营收 4.92 亿美元、企业用量连续 6 个月环比 +50%、公司亲口承认"我们超过 90% 的代码现在是由 Devin 写的"。客户名单上写着 Citi、Mercedes-Benz、Goldman Sachs、Elevance、Dell、Santander 与美国政府机构。这一轮把代码代理这条赛道从"能不能 work"的问题，推到"能不能继续翻倍"的问题。值得对照的是 —— 5 月 13 日 Anthropic 在 Code with Claude 给出 Managed Agents 的"做梦+多代理+Webhook"组合拳；5 月 14 日 xAI 推出 Grok Build CLI，并把发布节奏改成日更。三家头部代理产品在 2 周里同时升档，市场愿意为"能在企业里自己跑起来的代码"出溢价 —— 而 Devin 是这群里把数字晒得最透的一家。

🏛️ ERP 联姻 · SAP × ANTHROPIC

SAP Sapphire 宣布 Claude 经 MCP 接进 S/4HANA、SuccessFactors、Ariba

5 月 13 日 SAP 在 Sapphire 大会上揭幕「Autonomous Enterprise」战略，把 Claude 通过 Model Context Protocol（MCP）接进 SAP Business AI Platform，并经 Joule 入口接入 S/4HANA、SuccessFactors、Ariba。SAP 用 200+ 业务代理覆盖财务结账、HR 答询、采购订单改单、跨系统报销等具体动作；每一次代理动手，都仍在 SAP 已有的审批、政策、合规框架内执行。SAP 把 ERP 客户数（数十万）和 Anthropic 把 Claude 的推理能力放在一张表上：前者出"系统能改什么"的范围，后者出"应当怎么改"的判断。这是 Claude 与 SAP 此前在 SuccessFactors 上小范围接入的延续 —— 一年内从 SaaS 子模块，扩到整条 ERP 主干。

Claude Managed Agents 新增"做梦+多代理编排+结果回调+Webhook"

开源代理 · 军备赛

代理的故事到下半年开始分流 —— 有人去抢估值，有人去抢 GitHub 星，有人去发模型版本号。Hermes Agent 把开源代理的增长曲线再次抬上一个台阶，Grok V9 在 GPU 上跑完，arXiv 还在给"多代理 RL"补理论 —— 三条线，同一句话：代理是 2026 的标的，没人想错过。

🚀 开源代理 · HERMES AGENT

Hermes Agent 12 周冲到 ~16 万 GitHub 星 · 2026 最快开源代理框架

2 月 25 日上线的 Nous Research Hermes Agent，到 5 月已累计约 16 万 GitHub 星 —— 12 周内的增速比 OpenClaw 早期还快，成为 2026 上半年增长最快的开源代理框架。Hermes 的卖点不是"用了哪家模型"，而是"自我进化"：每完成一个任务，代理自我生成偏好对 + 经验回放，写回本地长期记忆，下一只代理可读取继承。配套的 v2026.5.16 版本继续打磨"代理之间共享技能"的协议。和 Cognition、Devin 走"闭源 API + 企业合同"路线相反，Hermes 押的是开源代理的群体效应：当任何人都能在本地跑出一只"会自我修复"的代理，闭源代理就必须靠场景与合规继续往企业里挖。

📄 arXiv · 2605.02801

"编排痕迹"做 RL · 给多代理系统补一个能学的奖励信号

5 月底 arXiv 2605.02801 提出 Orchestration Traces：把多代理系统的执行过程看作时间序列图 —— 子代理生成、任务派发、跨代理通信、工具调用、结果聚合、终止判定都记录为可学习事件，进而用 RL 直接在"编排痕迹"上做策略优化。这把上半年三类工作做了汇总：Anthropic 这周给出的"做梦+多代理"是工程基础设施；Cognition 给出的"90% 自家代码"是现实产物；这份工作给出的是"它们到底学到了什么"的可解释、可奖励信号。读法上，把它叠加在前期"Memory-R1"、"TriMem 三层记忆"系列一起看 —— 代理这一年走完的不是模型尺寸的曲线，而是"过程 / 记忆 / 学习"三件事的并发曲线。
